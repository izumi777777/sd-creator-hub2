# 【機能】キャラクターと作品を登録する理由【下書き】

**種別:** 機能説明（利用者向け）  
**関連:** [technical_character_work.md](technical_character_work.md)

---

## この画面でできること

- **キャラクター:** 制作で使うキャラの単位（名前・メモ・紐づくプロンプトやストーリーの起点）。
- **作品:** シリーズや企画の束（キャラと関連付け、後からストーリーや画像の整理に使う）。

SD の CKPT / LoRA はローカル前提でも、**ポータル上の「誰の話か」「どの企画か」** を固定しておくと、ストーリー生成や画像一覧の迷子が減ります。

---

## 使い方のコツ

- 先に **キャラ** を作り、**作品** を紐づけると、ストーリー画面で選択しやすいです。
- メモ欄は **自分が後で検索しやすい言葉**（通称、モデル名のメモなど）でよいです。秘密情報は書かない。

---

## おわりに

マスタデータなので項目は少なめでも、**一貫した ID で他機能が参照**できるのが効きます。
