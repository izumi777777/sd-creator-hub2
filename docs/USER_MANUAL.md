# Creator Portal 使い方マニュアル

SD 制作・販売まわりをまとめるローカル／自前サーバ向けの Web アプリです。画面上部のナビから各機能に移動します。

**画面下部の「Gemini に相談」**から、生成 AI・Pixiv・販売などについて自由に質問できます（表示中のページに応じて相談モードの初期値が切り替わります。プルダウンで手動変更も可）。**同じブラウザタブでは直前のやりとりがセッションに残り**、続きの質問に文脈を踏まえて答えます。長すぎると古いターンから捨てます。**会話をリセット**で履歴を消せます。相談モードを変えて送信すると自動で履歴はクリアされます。

**ストーリーと連携:** **保存済みストーリーの詳細ページ**では、本文・シーン・プロンプト抜粋が「ストーリー資料」として相談に添付できます（チェックでオン／オフ）。資料があるときは相談モードの初期値が **販売文・キャプション（Pixiv 等）** になります。**一覧で生成した直後**は「この生成結果を相談ドックに渡す」でセッションに載せてから、下部の相談から Pixiv 用のタイトル・キャプション・タグを依頼できます。

## 1. 動作環境とセットアップ

### 必要なもの

- Python 3.10 以上推奨
- （Gemini 利用時）[Google AI Studio](https://aistudio.google.com/) の API キー
- （画像アップロード・エクスポート利用時）AWS S3 のバケットと認証情報

### インストール

リポジトリ直下で仮想環境を有効化し、依存関係を入れます。

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### 環境変数（`.env`）

リポジトリ直下に `.env` を置きます（`.gitignore` で除外済み）。主な変数は次のとおりです。

| 変数 | 説明 |
|------|------|
| `FLASK_SECRET_KEY` | 本番では必ずランダムな値に変更 |
| `DATABASE_URL` | 未指定時は SQLite `sqlite:///creator_portal.db` |
| `GEMINI_API_KEY` または `GOOGLE_API_KEY` | Gemini 呼び出し用 |
| `GEMINI_MODEL` | 既定は `gemini-3-flash-preview`（軽量）。ブラウザの **gemini.google.com で使っている Gemini 3.1 Pro** に API でも近づけるなら `gemini-3.1-pro-preview`（[モデル一覧](https://ai.google.dev/gemini-api/docs/models)の表記どおり）。Pro は Flash よりコスト・レートが重くなりがちです。 |
| `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` | S3 利用時 |
| `AWS_S3_BUCKET` / `AWS_S3_REGION` | バケット名・リージョン |
| `SD_WEBUI_BASE_URL` | （任意）AUTOMATIC1111 のベース URL。例: `http://127.0.0.1:7860` または EC2 の `http://<パブリックIP>:7860`。設定するとストーリー詳細から章プロンプトで **txt2img → S3（original + stripped）** が使えます（Web UI は `--api` で起動していること）。 |
| `SD_WEBUI_TIMEOUT` | （任意）API 待ち秒。既定 `600`。 |
| `SD_WEBUI_DEFAULT_CHECKPOINT` | （任意）キャラの「使用SDモデル」が空のときのチェックポイントファイル名（例: `waiIllustriousSDXL_v160.safetensors`）。 |
| `SD_SCHEDULER_ENABLED` | `1` / `true` / `yes` のとき、起動中のプロセス内で**バックグラウンド**が定期的に予約ジョブを実行します（寝ている間に生成させるには、**常時起動のマシンまたは VPS** でポータルを動かし続ける必要があります）。 |
| `SD_SCHEDULER_POLL_SECONDS` | 予約チェック間隔（秒）。既定 `60`。15～3600。 |
| `PORT` または `FLASK_RUN_PORT` | `python run.py` の待ち受けポート（未指定時は **5050**。例: `5000` に戻す） |
| `FLASK_RUN_HOST` | バインドアドレス（既定 `127.0.0.1`） |
| `LOG_LEVEL` | ログの詳しさ（既定 `INFO`。Gemini のモデル試行・成功／失敗は INFO／WARNING でコンソールに出る） |

### データベース

**`python run.py` で起動する場合**、起動直前に `flask db upgrade` と同じ処理が**自動で**走ります（`works` などのテーブルが無いエラーを防ぐため）。  
自動実行を止めたいときは環境変数 `SKIP_AUTO_MIGRATE=1` を設定してください。

**`flask run` や gunicorn だけ**で動かす場合は、初回・更新後に手動でマイグレーションが必要です。

```bash
set FLASK_APP=run:app
flask db upgrade
```

### 起動

開発用（マイグレーション自動＋デバッグサーバ）:

```bash
python run.py
```

Flask 標準（ポートも 5050 に合わせる例）:

```bash
set FLASK_APP=run:app
set FLASK_RUN_PORT=5050
flask run
```

ブラウザで `http://127.0.0.1:5050` を開きます（`PORT` を変えた場合はその番号に合わせてください）。

**画像生成の予約（夜間バッチ）:** ストーリー詳細で「予約登録」すると DB にキューが入ります。**予約どおりに動かすには** `SD_SCHEDULER_ENABLED=1` でポータルを起動したままにしてください。`gunicorn` で複数ワーカー（`-w 2` 以上）にするとスケジューラが二重起動しうるため、予約運用時は **`-w 1`** を推奨します（またはスケジューラを別プロセスに分ける等の運用）。

### CLI（任意）

既定キャラクターをまとめて登録する例:

```bash
flask seed-characters
```

## 2. 画面別の使い方

### ダッシュボード

概要のリンク集です。作業の起点にしてください。

### キャラ

キャラクターマスタ（名前・タグ・SD モデル・**LoRA 名**・絵文字など）を登録・編集します。  
**ストーリー生成**や**プロンプトライブラリ**では、ここで選んだキャラに紐づきます。ストーリーで LoRA 行を出したい場合は **LoRA 名**を実際のトリガー文字列に合わせておくとよいです。

### 作品

作品（タイトル・ステータス・価格・販売 URL など）を管理します。**画像**登録時に作品へ紐付けできます。

### プロンプト（ライブラリ）

キャラごとの Positive / Negative・シチュ名・メモを保存します。**ストーリー生成**で複数選択すると、ベースプロンプトとして Gemini に渡せます。

### ストーリー

- **キャラ**・**あらすじ／依頼**・**手入力ベース（Positive）**・**手入力ベース（Negative）**・**ライブラリ複数選択**のうち、**いずれか 1 つ以上**が必須です（キャラは常に必須）。共通ネガは各章の `neg` に反映するよう Gemini に渡します。
- キャラを選ぶと、ライブラリの候補がそのキャラに絞り込まれます（Ctrl で複数選択）。
- **生成する**で Gemini が JSON を返し、右ペインに本文・共通設定・シーン別プロンプト（パターン複数可）が表示されます。
- **この内容を DB に保存**でタイトル・概要・長文本文・章 JSON などを保存します。
- **加筆・修正（AI）**: 生成結果の下、または保存済みストーリーの**詳細ページ**上部から、自然言語で指示（例:「シーン3のあとに◯◯を追加」）を送れます。htmx でプレビューが更新され、**上書き保存**で DB を更新できます（保存済みのときは詳細画面からの修正プレビュー経由で上書き）。
- **Pixiv 投稿文案**: 詳細ページの「Pixiv 投稿文案」で、Gemini 相談の回答を貼り **Pixiv 向けに整形**すると `*` や `>` などの Markdown 記号を除いたテキストにできます。タイトル・キャプション・タグを編集して **保存**するとストーリーに紐づけて DB に保持されます。
- **加筆・修正と手動プロンプト**: 露骨な成人向けの描写などは Gemini API の安全フィルタで拒否されることがあります。その場合は **Stable Diffusion 側でプロンプトを調整**し、詳細ページの **「シーンにプロンプトを手動追加」** で該当シーンに `prompt_variants` を追記するか、**「章データ全体を JSON で編集」** で `chapters_json` を直接更新してください。

**注意:** 長文・多シーンはトークン量が増えます。API のレート制限やクォータに達した場合は、しばらく待つか [Gemini のレート制限](https://ai.google.dev/gemini-api/docs/rate-limits)・請求設定を確認してください。`.env` の `GEMINI_MODEL` を別モデルに変えると改善することがあります。

### テキスト生成

Pixiv / DLsite / PictSpace 向けの販売テキスト案を、作品概要とキャラ情報から Gemini で生成します（htmx で結果表示）。

### 画像

ローカル画像を **S3 にアップロード**し、DB に登録します。**ファイル選択で複数枚まとめて**（PNG / JPEG 等、そのままの形式で）アップロードできます。キャラクターごとに **メタデータあり（`original`）** と **メタデータなし・配布用（`stripped`）** のサブフォルダを選べます。**ストーリー**に紐づけて保存すると、ストーリー詳細ページのギャラリーと画像一覧の絞り込みに表示されます。非公開バケットの場合は、一覧やエクスポートで**署名付き URL**が使われます。キャラ・作品・ストーリー・フォルダ種別で絞り込みできます。

### メタデータ削除

ローカルフォルダ内の画像から EXIF 等を削り、指定ルールで別フォルダへ振り分けます。サーバ上のパスを入力して実行します（バッチ処理）。

### PDF/ZIP

DB に登録済みで S3 と紐づく画像を選び、**PDF** または **ZIP** をダウンロードします。

### 売上

月次の売上・フォロワーなどを記録・一覧します。

## 3. トラブルシューティング（よくあるもの）

| 現象 | 確認すること |
|------|----------------|
| ストーリー生成が 429 やクォータエラー | API キーのプロジェクト枠、`GEMINI_MODEL`、時間をおいて再試行 |
| 生成結果が表示されない（htmx） | ブラウザの開発者ツールでネットワークエラー確認。4xx/5xx も画面に出るようになっています |
| S3 アップロード失敗 | 認証情報・バケット名・リージョン・バケットポリシー |
| マイグレーションエラー | `FLASK_APP=run:app` が通るか、DB ファイルのロック |
